# 2D肉鸽地牢游戏

**语言**: 中文 | [English](https://github.com/Aerhuo/RoguelikeDungeon_2DGame)

![C++](https://img.shields.io/badge/C++-17%2B-00599C?logo=c%2B%2B) ![SFML](https://img.shields.io/badge/SFML-2.6-8CC445?logo=sfml) ![CMake](https://img.shields.io/badge/CMake-3.10%2B-064F8C?logo=cmake) ![OOP](https://img.shields.io/badge/Architecture-OOP%20%26%20Event%20Queue-ff69b4)

**RoguelikeDungeon_2DGame** 是一款基于现代 C++ 与 SFML(简单快速多媒体库) 构建的轻量级 2D 回合制地牢探险游戏。本项目由 Aerhuo 发起，旨在通过完整的游戏开发实践，深入探索并应用面向对象编程的核心架构思想，实现一个具备扎实底层架构与基础策略深度的 Roguelike 游戏雏形。

> ⚡**核心架构优势**：采用高内聚低耦合的现代游戏引擎设计理念。
> 1. **双层网格架构**：将静态地形与动态实体在数据结构上剥离，碰撞检测达到 $O(1)$ 时间复杂度。
> 2. **事件队列驱动**：引入命令模式，分离输入决策与物理执行，解决回合制结算的时序重叠问题。
> 3. **全局上下文**：构建统一的 `World` 结构体，消除类之间的循环依赖。
> 4. **内存安全**：全面应用 `std::unique_ptr` 独占智能指针与延迟删除机制，避免内存泄漏与迭代器失效。

## 🚀 核心特性

### 🧠 纯正的面向对象设计

* **多态实体**：基于基类 `Entity` 派生 `Player` 与多种 `Enemy`，共享数据结构与渲染管理器。
* **开闭原则**：新增怪物种类或事件类型时，无需修改核心的 `Game` 主循环逻辑。

### 🗺️ 双层网格系统

* **地形层**：负责记录不可变的墙壁与可通行的地板，承载基础迷宫生成算法。
* **实体层**：利用二维指针数组精确记录每一格上的活动对象，实现高效率的阵营判定与战斗触发。

### ⏳ 事件驱动回合制

* **意图预判**：实体决策不再直接改变物理坐标，而是生成 `MOVE` 或 `ATTACK` 等意图事件。
* **统一解算**：所有指令被推入 `std::queue<Event>`，由游戏主循环严格按照时序依次解算，为后续动画演出预留空间。

### 🔍 算法驱动的智能行为

* **流场寻路**：利用 BFS 广度优先搜索结合层序遍历，一次性计算全图怪物到达玩家的最短距离矩阵。
* **即时响应**：怪物根据 `dist` 距离矩阵动态规划路线，实现高效的路径追击。

### ⚔️ 极致的性能优化

* **极速删除机制**：在清理死亡怪物时，采用与动态数组末尾元素交换并弹出的 Swap and Pop 算法，将删除开销降至极限的 $O(1)$。

## 🛠 技术架构

| 层级 | 技术组件 | 职责描述 |
| --- | --- | --- |
| **渲染层** | SFML | 负责窗口创建、按键轮询、帧率控制与图形绘制。 |
| **控制层** | `Game` 类 | 持有主循环，统筹 `World` 数据与视图摄像机。 |
| **逻辑层** | 事件队列 | 接收实体指令，处理碰撞、移动逻辑与伤害公式结算。 |
| **实体层** | `Player`, `Slime` 等 | 继承自 `Entity`，独立处理输入判断或 AI 决策。 |
| **数据层** | `World`, `Map` | 封装距离矩阵、地形网格、实体指针网格，提供纯粹的数据上下文。 |

## 🛠️ 开发准备

### 环境要求

* **C++ 编译器**: 支持 C++ 17 或更高版本 (如 GCC 9+, Clang 10+, MSVC 2019+)。
* **构建工具**: CMake 3.15 或更高版本。
* **多媒体库**: SFML 2.6.x (需添加至系统环境变量或正确配置 CMake 链接路径)。

### 编译与运行

本项目使用 CMake 进行构建管理：

```bash
# 1. 创建构建目录并进入
mkdir build
cd build

# 2. 生成构建文件
cmake ..

# 3. 编译项目
cmake --build .

# 4. 运行游戏
# Windows 环境下 (可执行文件直接生成在 build 目录中)：
./MyGame.exe
# Linux / macOS 环境下：
./MyGame

```
